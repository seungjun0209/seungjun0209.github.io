## CSS 선택자에 대해서
이번에는 css에 대해서 수업을 듣다가 새로 알게 된 내용을 정리하려고 합니다.

#### 속성 선택자
특정 속성을 가진 태그만 따로 css를 먹일 수 있다. 주로 form 태그 관련 대상에 사용한다.
```css
input[type="text"] { background-color: yellow; } /* 이건 확실하게 기억하기!!! 타입이 text일 때, 적용*/
input[type^="t"]   { background-color: yellow; }
input[type$="xt"]  { background-color: yellow; }
input[type*="ex"]  { background-color: yellow; }
```

#### 하위 선택자
공백을 사용해서 표시한다. 특정 태그 안에 있는 후손들은 전부 css를 먹는다. 아래 있는 코드를 예시로 들면 div 안에 있는 p 안에 있는 모든 a 태그들은 다 css를 먹는다.
```css
div p a { background-color:yellow; }
```

#### 자식 선택자
꺽쇄(>)를 사용해서 표시한다. 특정 태그의 직계 자식 태그만 css를 먹는다. 당연히 .index처럼 클래스로도 표시할 수 있다.
```css
h1 > em { background-color:yellow; }
h1 > .index { background-color:yellow; }
```

#### 가상 선택자
현재 상태에 따라서 css를 먹일 수 있다. 나는 이걸 javascript를 사용하는 이벤트와 비슷하다고 생각했다. 그러나 hover를 사용해서 잠깐 위치를 바꾼다던지 그런 효과가 마치 이벤트가 발생한 것처럼 보이지만 특정 상황에 다른 css를 먹어서 바뀌는거지, 이벤트는 아니다. 결론 -> 그냥 css
```css
button:hover { transform: scale(1.03); } →버튼에 마우스가 올라갔을 때
input:disabled{ outline: 2px solid; } →비활성화 된 상태일 때
```

#### em
부모 노드에 있는 font-size를 기준으로 상대적으로 폰트 크기를 바꿀 수 있다. 예시) 부모 노드의 font-size가 20px이라면 자식 노드에서 1.5em을 했을 때, 자식 노드의 font-size는 20*1.5px로 30px의 크기를 가지게 된다.
